# 唯一入库路径重构、库外编辑拦截、导航镜像加速与有序动态迁移 —— Analysis_Modification_Plan-116.md

## 1. 任务背景
为了实现数据库准入的物理绝对一致性，用户要求重构入库架构：入库动作由且仅由 USN Journal 触发。系统需提供视图级拦截提示、自动镜像加速加载，并提供以“库根目录”为首、辅以“最近访问”文件夹的有序迁移目标。

## 2. 问题定位
- **入库权限越位**：UI 层具备的主动入库权限（`registerItem`）导致物理状态与数据库可能脱节。
- **编辑权限盲区**：在物理路径导航时，用户可对库外项目进行元数据操作，导致“幽灵记录”。
- **性能冗余**：物理导航进入托管库时未利用 SCCH 内存镜像，加载缓慢。
- **迁移体验不足**：原右键菜单未对迁移目标进行优先级排序，不便于用户快速定位常用目录。

## 4. 详细解决方案

### 4.1 数据库持久化权限底层收拢
- **锁定 INSERT 接口**：修改 `MetadataManager::persistAsync`，增加来源校验。除 `AutoImportManager`（USN 线程）外的入库请求一律拦截。
- **清理主动逻辑**：废除 UI 命令中所有对 `registerItem` 的调用。

### 4.2 迁移功能的物理化（原“收揽入库”）
- **职责纯化**：`ImportHelper::importPaths` 仅负责物理层面的移动（`MoveFile`）。移动完成后即告结束。
- **异步响应**：入库动作完全由 USN 捕获后由后台异步执行。

### 4.3 视图级编辑权限拦截
- **拦截准则**：在内容面板 Setter 中，若 DataSource = nav 且当前路径在库外，拦截操作。
- **提示反馈**：调用 `FramelessMessageBox` 提示用户先执行迁移。

### 4.4 视图唯一性判定逻辑
- **技术路径**：通过内容面板当前的 `DataSource` 和 `m_currentPath` 进行高性能权限判定。

### 4.5 导航模式自动镜像重定向（加速加载）
- **自动切换**：在 `loadDirectory` 触发时，若目标路径属于库内，停止物理扫描。
- **镜像渲染**：从 `MetadataManager` 哈希表中过滤出该前缀下的项并渲染，实现秒开且自动携带元数据。

### 4.6 有序动态迁移算法
- **菜单排序逻辑（核心）**：
    1. **固定首项**：获取当前盘符托管库根目录 `ArcMeta.Library_盘符`，作为菜单第一位。
    2. **最近访问项**：执行 SQL 检索当前盘符库内 `is_folder=1` 且 `is_trash=0` 的记录，按 `atime` 降序排列。
    3. **数量控制**：取前 14 个最近访问目录，排在首项之后。
- **菜单交互**：
    - 主项：“**迁移**”（点击默认迁移至首位目录）。
    - 二级菜单：“**迁移到活跃文件夹...**” -> 展示上述有序列表。

### 4.7 AGENTS.md 规则预置
在 `AGENTS.md` 中固化如下条款：
- **[唯一入库路径红线]**：禁止 UI 直接入库。
- **[导航加速规范]**：入库文件夹导航必须自动切换镜像模式。
- **[有序迁移规范]**：迁移目标必须以库根目录为首，后续跟随 14 个最近访问目录。
