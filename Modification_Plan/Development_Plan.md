# ArcMeta 项目开发规约与演进计划 —— Development_Plan.md

## 1. 托管库 (ArcMeta.Library_) 自动化链路规约

### 1.1 监控架构演进：恢复 NativeFolderWatcher (IOCP) 主轨
- **唯一监控信源**：彻底废除并卸载 USN Journal 监控。系统物理监控逻辑重新全部收拢至基于 `ReadDirectoryChangesW` 的 **NativeFolderWatcher (IOCP)** 机制。
- **作用域过滤与移除调试**：
    - **过滤策略**：监控 `ArcMeta.Library_[盘符]` 文件夹及其所有子树的物理变动。
    - **彻底移除 QMessageBox 调试日志**：由于全面弃用并废除 USN 机制，必须彻底移除 `AutoImportManager.cpp` 等中由于原先 USN 调试目的而引入的 `QMessageBox` 弹出框调试、`safeShowMessageBox` 全局辅助函数、以及相关调试信息及引用的头文件，确保不留任何痕迹。
- **入库触发机制**：元数据登记与摄取动作由 `NativeFolderWatcher` 在捕获到文件系统物理变动后，通过后台异步线程反向回调 `MetadataManager::registerItemsAsync` 或 `markAsRegistered` 触发。

### 1.2 镜像铁律：1:1 物理复刻 (Absolute Mirroring)
- **物理决定逻辑**：在 `ArcMeta.Library_` 分支下，侧边栏分类树必须是磁盘物理目录的 **1:1 绝对镜像**。
- **严禁逻辑脑补**：
    - 文件夹的深度、层级、父子关系必须与 Windows 文件系统保持原子化强一致。
    - 严禁任何形式的“虚拟分类”、“逻辑重组”或“跨盘符聚合”。
    - 任何逻辑层面的重命名、删除、移动操作必须同步镜像到物理磁盘；反之，物理磁盘的变动必须无条件驱动逻辑层的刷新。

### 1.3 身份登记与摄取流程
- **身份登记 (Status 0)**：
    - 监听到变动后，系统应立即调用注册函数。
    - 将项目的 **File_ID** 与**传统物理路径**写入 `metadata` 表。
    - 物理标记 `ingestion_status`（标记值）初始化为 **0**。
- **解析与完工 (Status 1)**：
    - 登记完成后，调用 `ImportHelper.cpp` 作为统一解析引擎，处理所有标记为 **0** 的项目。
    - **状态更新**：解析任务（颜色、属性等）彻底完成后，将数据库标记值从 **0 更新为 1**。

## 2. 手动干预与精准同步规约

### 2.1 右键菜单交互逻辑
- **对象范围**：凡是物理路径位于 `ArcMeta.Library_` 文件夹内的项目。
- **“重新扫描”显示规则**：
    - **强制呈现**：无论该项目是否已经入库（即不论标记值是 0、1 还是未入库状态），右键菜单中必须**始终显示“重新扫描”**选项。
- **核心原则：作用域可控**。
- **执行范围**：严格锁定为 UI 界面中**“被选中”**的项目（文件或文件夹子树）。
- **行为定义**：
    - 强制执行物理状态同步与元数据重新解析。
    - **全量扫描定义**：盘符栏下的“重新扫描该盘”是指扫描该盘符下的“ArcMeta.Library_[盘符]”文件夹里所有的数据（对应用户原话：““重新扫描该盘” 应该是扫描该盘符下的“ArcMeta.Library_[盘符]”文件夹里所有的数据”）。
    - **状态无关性**：无论被选中项（或全量文件夹）当前的标记值是 0 还是 1，系统必须**无视现有状态**，强制执行重新解析流程。
- **严禁扩权**：禁止越过“被选中”蓝框去触碰任何非选中区域的数据。

### 2.2 拖拽入库冲突拦截
- **触发场景**：用户将位于 `ArcMeta.Library_` 路径下的项目拖拽至侧边栏分类。
- **拦截逻辑**：
    - **已入库判定**：若该项目已经入库（标记值为 1），必须立即阻断操作。
    - **交互反馈**：弹出显式警告提示：**“该项目已入库，无需再次入库”**。
- **执行逻辑**：只有在未入库的情况下，才允许执行重新扫描并入库的操作。

## 3. 视觉反馈与性能加固规约 (进度弧线)

### 3.1 进度弧线驱动逻辑
- **标记驱动而非扫描驱动**：进度弧线（百分比圆圈）必须完全通过数据库中的 **0 (待解析)** 和 **1 (已完成)** 标记值计算得出。
- **严禁实时物理统计**：计算进度时，禁止调用任何物理读盘函数（如 `QDirIterator`）。
- **公式**：`进度 = (该目录下状态为 1 的项目数) / (该目录下状态为 0 和 1 的项目总数)`。

### 3.2 数据库持久化与加载
- **物理落地**：文件夹的进度百分比必须记录到数据库的专属字段中。
- **启动零开销**：下次启动主程序时，UI 直接从数据库加载已存的进度数值进行渲染。严禁在启动阶段反反复复进行聚合计算。

### 3.3 逻辑收拢：专属原子函数
- **职责唯一性**：必须建立专属函数负责标记值的更新与比例值的同步。
- **数据一致性**：确保每一笔 0 到 1 的翻转均由该专属函数原子化完成，并同步更新父目录的持久化进度值。

## 4. 架构红线与禁令

- **严禁重复造轮子**：全应用禁止出现两套平行的扫描入库引擎。`AutoImportManager` 仅作为变动监听的触发器，具体的解析逻辑必须收拢至 `ImportHelper`（或归一化后的摄取模块）。
- **严禁启动自扫描**：主程序启动时严禁自动执行全量物理递归。一切物理读盘开销必须由 “NativeFolderWatcher (IOCP)” 物理变动信号或“用户手动选中”驱动，以确保 UI 绝对流畅，不发生假死与卡顿。
- **彻底废除及移除“失效数据”机制**：
  - **分类移除**：必须彻底从侧边栏系统分类树（`CategoryModel`、`CategoryPanel` 等）中剔除 “失效数据” (invalid_data) 节点。
  - **逻辑与视图清理**：完全清理其底层在 `MainWindow` 中的实例、显隐控制逻辑，并物理删除/剔除 `InvalidDataListView.h`、`InvalidDataModel.h` 对应的前端视图和模型文件，使整个工程无任何残留。

## 5. 盘符栏 (Drive Bar) 逻辑架构规约

### 5.1 状态联动逻辑
- **任务执行态 (Running)**：当 `ImportHelper.cpp` 执行文件迁移或解析任务时，目标路径所属盘符的按钮必须实时切换至 **Running** 状态（即界面显示为转圈动画）。
- **任务结束态**：任务完成后，按钮应根据托管文件夹是否存在恢复至 **Active** 或 **Inactive** 状态。

### 5.2 交互引导逻辑
- **激活前拦截**：对于尚未创建“ArcMeta.Library_[盘符]”托管文件夹的盘符（即处于 **Inactive** 状态的按钮），用户左键单击时，系统不得执行空跳转。
- **提示语规范**：拦截后必须弹出提示：“**尚未创建文件夹，请单击右键进行创建托管文件夹**”（对应用户原话）。

## 7. 托管库内存模式秒开与性能重构规约

### 7.1 零 I/O 元数据构建与空判定
- **元数据零物理 I/O 提取**：在内存镜像模式下，构建 `ItemRecord` 时仅允许利用内存中已缓存的 `RuntimeMeta` 属性，彻底禁止调用 `fetchWinApiMetadataDirect` 进行物理句柄轮询。
- **空判定规约**：废除利用 `QDir::entryList` 对子文件夹进行物理判空的逻辑。子文件夹是否为空必须通过读取内存中的树级倒排索引数据进行判定。

### 7.2 树级目录索引与检索加速
- **引入快速树级索引**：在 `MetadataManager` 中维护 `m_parentToChildren` 倒排哈希，将目录子项检索开销降为 $O(1)$。
- **消灭全量线性遍历**：在 `loadDirectory` 重定向分支中，绝对禁止线性遍历扁平缓存表，必须通过父子索引进行直接提取。
- **锁粒度控制**：缩短 `m_mutex` 共享读锁的占用周期，拷贝出子路径列表后立刻释放锁，杜绝前后台读写锁阻塞。

### 7.3 进度持久化与加载优化
- **消灭循环内 SQL 查询**：禁止在列表构建循环中针对每个子文件夹单独触发 SQL 预编译和查询，进度百分比必须从进度内存缓存 `m_folderProgressCache` 中读取。

### 7.4 目录导航去耦合与物理扫描回归
- **物理回归**：彻底取消从“目录导航”进入 `ArcMeta.Library_[盘符]` 时自动切换为“内存镜像加载”的分流加速逻辑。
- **职责清晰**：
  - 磁盘路径模式（物理导航）始终保持使用 **“物理扫描模式”**，利用后台线程池真实扫描并加载物理磁盘目录中的直接子项和子树，不再读取内存镜像数据。
  - 侧边栏分类模式继续由 **DB 驱动（SQLite 内存模式）** 高速加载。
  - 两套驱动机制完全解耦，各自独立执行自己的业务逻辑。

## 9. “添加至收藏夹”右键菜单功能开发规约

### 9.1 多选与多项支持
- **批量收藏**：若用户在内容容器（`ContentPanel`）中多选了多个有效项目，点击右键菜单的“添加至收藏夹”时，应依次将所有选中项目的物理路径（`PathRole`）全部添加至收藏夹中。
- **项目类型**：无论是文件还是文件夹，均应支持添加至收藏夹。

### 9.2 全局适用性
- **始终可见**：不论内容容器当前处于何种视图（无论是在“物理目录导航”还是“侧边栏分类模式”下），只要选中项为有效物理路径（且非空），右键菜单即始终显示“添加至收藏夹”选项。

### 9.3 自动排重
- **不置灰、静默排重**：为避免频繁进行 UI 扫描检查降低流畅度，该选项在右键菜单中照常高亮显示（不置灰）。点击添加时，由 `NavPanel` 底层的排重机制自动进行去重判定，对已收藏的项目静默忽略。

### 9.4 友好气泡提示
- **气泡反馈**：添加成功后，在用户鼠标指针所在位置弹出淡入淡出的气泡提示（内容为：“已成功添加至收藏夹”），使用系统的 `ToolTipOverlay::instance()->showText(...)` 展现。

## 10. 侧边栏分类图标自定义规约 (Category Icon Customization)

### 10.1 数据字典与持久化拓展
- **底层数据库字段拓展**：在 `categories` 表结构中追加 TEXT 类型字段 `icon`，默认值为空或 `"folder_filled"`。
- **实体属性定义**：在 `struct Category` 结构体中新增 `std::wstring icon;` 字段，并在 `CategoryRepo.cpp` 中处理其插入、更新与查询绑定逻辑。
- **动态升级迁移**：在 `DatabaseManager::loadDb` 函数中引入 categories 表 table_info 探测与动态 ALTER COLUMN 机制，确保启动时自动升级历史数据库文件、零损无感适配。

### 10.2 右键菜单交互体系
- **菜单位置与入口**：在侧边栏分类项右键弹出菜单（`CategoryPanel::setupContextMenu`）中新增主选项：“**文件夹图标**”（对应用户原话：“在右键菜单中新增一个主选项”）。
- **子菜单 SVG 展现**：主选项下挂载一个二级子菜单（`QMenu`）。该子菜单中平铺或分页承载系统内置的所有 SVG 矢量图标（SVG 数据统一从 `SvgIcons.h` 中检索加载），子项展示对应矢量图标。
- **操作与动态反馈**：用户点击子菜单中的 SVG 图标后，立即触发 `CategoryRepo::update` 将变更写入本地磁盘 SQLite 数据库并调用 `CategoryModel::refresh()` 重新加载，实现侧边栏图标无缝实时变更新渲染。

## 11. 盘符栏自定义文件夹外观自定义规约 (Drive Bar Folder Button Customization)

### 11.1 持久化结构设计
- **去中心化 KV 绑定**：由于自定义文件夹路径在 `AppConfig` 的 `"DriveBar/CustomMonitoredFolders"` 中以扁平 `QStringList` 的形式存取，为了绑定对应的个性化外观属性，需在 `AppConfig` 下追加如下绑定：
  - 按钮颜色设定：`DriveBar/FolderColor_[物理路径]` (类型为 `QString`, 存 `#RRGGBB` 格式色值)
  - 按钮图标设定：`DriveBar/FolderIcon_[物理路径]` (类型为 `QString`, 存 `SvgIcons.h` 中对应的 SVG 键名)
- **数据清理同步**：当用户在盘符栏点击 “移除” 该自定义文件夹时，系统必须在清理 `CustomMonitoredFolders` 的同时，静默将该物理路径对应的 `FolderColor_` 和 `FolderIcon_` 键名配置一并抹除，防止配置文件膨胀。

### 11.2 右键菜单交互体系对齐
- **主选项菜单项扩展**：在顶部盘符栏自定义监控按钮（`FolderButton`）右键弹出菜单中，在原有的 “新建自动导入” 与 “移除” 选项之下追加：
  - **“设置颜色”**：弹出 `FramelessColorPicker` 颜色选取器，选择后应用并写入配置。
  - **“随机颜色”**：从系统内置的调色板中随机抽取一个高亮色应用。
  - **“文件夹图标”**：平铺或分页承载内置 SVG 图标子菜单，允许选择并实时更新。
- **重绘与无缝视觉更新**：外观设定变更完成后，立即重新触发 `DriveBar` 局部重绘（调用对应的 `update()` 或重设样式），实现自定义文件夹按钮图标与颜色的瞬时、流畅更变渲染。
