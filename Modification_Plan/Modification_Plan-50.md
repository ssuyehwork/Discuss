# 列表视图表头列宽固定化及冗余代码删除重构 —— Modification_Plan-50.md

> 状态：已批准，执行中 / 已执行完成

## 1. 任务背景
在列表模式下，表头各列（状态、星级、颜色、标签、类型、大小、修改日期）此前支持拖拽交互。但随着应用排版设计的稳定，用户更倾向于将其列宽固定，使软件窗口缩放时由名称列（第 0 列）进行弹性自适应（Stretch）拉伸，而其余 7 列则保持磐石般的固定尺寸。将它们全部变更为 `QHeaderView::Fixed` 后，不仅能保证排版永远美观无错乱，还能大幅删掉用于处理防重叠、挤压和持久化复杂状态的代码，极力给系统瘦身。

## 2. 问题定位
修改范围处于 `src/ui/ContentPanel.cpp` 中的 `ContentPanel::initListView()` 函数内：
- 以前对 `restoreState` 的读取和调用属于冗余。
- 对 `sectionResized` 信号的连接属于冗余。
- 列 1~7 的 `ResizeMode` 从 `Interactive` 重构为 `Fixed`。

## 4. 详细解决方案

### 4.1 代码重构与瘦身：`src/ui/ContentPanel.cpp`

定位到 `ContentPanel::initListView()` 的表头设置段：
1. **模式锁定与固定像素列宽**：
   - 保留名称列的自适应：`header->setSectionResizeMode(0, QHeaderView::Stretch);`
   - 对 1~7 列分别重调 `resizeSection`：
     - 状态（列1）：`50` 像素
     - 星级（列2）：`120` 像素
     - 颜色（列3）：`50` 像素
     - 标签（列4）：`120` 像素
     - 类型（列5）：`80` 像素
     - 大小（列6）：`100` 像素
     - 修改日期（列7）：`150` 像素
   - 将这 7 列的拉伸模式设为 `QHeaderView::Fixed`。
2. **彻底移除冗余**：
   - 彻底删除 `AppConfig` 的 `ListHeaderState` 持久化保存和读取。
   - 彻底删除 `sectionResized` 信号连接与全部防挤压折叠计算逻辑。

## 6. 实现准则与预警【核心】
1. **防止界面闪烁或异常**：不再使用 `saveState` / `restoreState`，表头在首次创建并显示时即处于最优固定比例，且每次重新开启应用均能完美对位，毫无物理拖动产生的隐患。
2. **极其安全的删除**：本次删除的是完全解耦的内部回调代码，可以带来编译上极佳的安全性和代码整洁度。
