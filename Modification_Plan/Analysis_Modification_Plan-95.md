# 目录导航面板隐式双区拆分及文件/文件夹收藏功能 —— Analysis_Modification_Plan-95.md

## 1. 任务背景
用户期望对“目录导航”面板（NavPanel）进行重构，将其划分为上下两个隐式功能区：
1. **上方区**：保留现有的磁盘目录树。
2. **下方区**：新增“快捷收藏”区域，支持通过拖拽将常用的**文件夹**或**文件**固定于此，并实现点击快速导航/定位功能。

## 2. 问题定位
- **模块**：`src/ui/NavPanel.cpp`, `src/ui/NavPanel.h`
- **函数**：`NavPanel::initUi()`, `NavPanel::onTreeClicked()`
- **架构分析**：
  现有的 `NavPanel` 采用单一 `QVBoxLayout` 直接承载目录树容器。需将其改为包含两个子容器的布局，并为下方区域引入新的数据模型及拖拽处理逻辑。

## 4. 详细解决方案

### 4.1 UI 结构重构 (`NavPanel.cpp`)
1. **布局调整**：在 `NavPanel` 的 `m_mainLayout` 中引入一个 `QSplitter` 划分上下区。手柄样式设为透明且宽度为 1px，以实现“隐式拆分”效果。
2. **上方区（Disk Tree）**：将现有 `m_treeView` 迁移至 Splitter 的上半部分。
3. **下方区（Favorites）**：
   - 新增 `m_favoriteView` (基于 `DropTreeView` 扩展)。
   - **交互增强**：
     - **右键菜单**：添加“取消收藏”操作，点击后从模型中移除该项并更新持久化配置。
     - **手动排序**：设置 `setDragDropMode(QAbstractItemView::InternalMove)`，允许用户通过拖拽自由调整收藏项的先后顺序。

### 4.2 数据模型与持久化
1. **持久化格式**：在 `AppConfig` 的 `NavPanel/Favorites` 中存储一个 JSON 对象数组，每个对象包含：`path` (路径), `type` (file/folder), `order` (排序索引)。
2. **数据同步**：每次拖拽排序或删除项后，立即触发序列化并刷入 `AppConfig`。

### 4.3 导航逻辑设计
1. **收藏文件夹点击**：发射 `directorySelected(path)`。
2. **收藏文件点击**：
   - 调用 `unifiedNavigateTo(parentDir)` 跳转至所属目录。
   - 监听 `ContentPanel::loadFinished` 信号，通过 `m_gridView->setCurrentIndex` 高亮显示对应文件。

## 6. 实现准则与预警【核心】
1. **右键冲突**：确保下方区域的右键菜单仅在收藏项上有效，空白区域点击不应弹出取消选项。
2. **排序稳定性**：手动排序后的顺序必须在应用重启后完美还原，不能依赖文件系统的默认排序。
3. **拖拽兼容性**：支持从上方磁盘树直接拖拽文件夹到下方收藏区。
