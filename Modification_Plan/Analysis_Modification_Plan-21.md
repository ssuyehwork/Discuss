# 架构分析与 UI 规范方案 (Analysis_Modification_Plan-21)

本方案对 FERREX 应用程序中所有“双击触发”及“悬浮弹出”类面板的圆角设计与逻辑限制进行了全量排查与总结。

---

## 1. 弹出面板圆角设计全量统计

通过对 `MouseButtonDblClick`（触发交互）与 `Qt::Popup`（窗口属性）的关联分析，确认应用内的圆角设计遵循分层分级的原则：

### 1.1 双击下拉历史类面板 (核心规范：8px)
这类面板由地址栏或搜索框双击触发，具有高度的视觉一致性：

| 组件名称 | 触发方式 | 圆角像素 (Corner Radius) | 样式定义文件 |
| :--- | :--- | :--- | :--- |
| **地址历史面板** | 双击地址栏 | **8px** | `AddressHistoryPanel.cpp` |
| **搜索历史面板** | 双击搜索框 | **8px** | `SearchHistoryPanel.cpp` |
| **右键上下文菜单** | 右键点击 | **8px** | `UiHelper.h` (QMenu 样式) |

### 1.2 功能性悬浮面板 (差异化设计)
非“历史列表”类的功能面板根据其内容复杂度采用了不同的圆角：

*   **颜色选择器 (ColorPicker)**: **6px**。
    *   *设计意图*: 与主界面输入框（6px）保持视觉逻辑关联，强调其作为“输入工具”的属性。
*   **快速预览窗 (QuickLookWindow)**: **12px**。
    *   *设计意图*: 作为全屏/大面积覆盖的沉浸式组件，使用更大的圆角以弱化边缘切割感，提升高级感。
*   **对话框 (FramelessDialog/CategorySetPasswordDialog)**: **6px 或 8px**。
    *   *说明*: 普通按钮通常为 4px/8px，背景容器多为 6px。

---

## 2. 历史记录限制逻辑对比

虽然视觉圆角统一为 8px，但底层存储上限根据使用频率进行了差异化配置：

| 功能模块 | 记录上限 | 逻辑所在位置 | 限制策略 |
| :--- | :--- | :--- | :--- |
| **地址栏路径** | **15 条** | `AddressBar::saveToHistory` | 针对高频访问的物理路径，上限较宽。 |
| **搜索关键词** | **10 条** | `MainWindow.cpp` (doSearch) | 针对临时搜索意图，上限较紧，保持列表清爽。 |

---

## 3. 架构结论与建议

1.  **“双击下拉面板 = 8px”是既定规范**：排查结果显示，所有通过双击输入框触发的历史下拉列表确实统一采用了 **8px** 圆角。这种设计通过一致的物理反馈，帮助用户快速识别“这是我的历史足迹”。
2.  **记录限制的合理性**：当前地址栏 15 条、搜索框 10 条的限制是合理的。地址路径具有更强的持久访问价值，而搜索关键词通常具有时效性。
3.  **未来的统一化方向**：
    *   建议将历史记录的 `8px` 和输入框的 `6px` 定义为全局常量（如 `CORNER_RADIUS_INPUT = 6` 和 `CORNER_RADIUS_POPUP = 8`），以防止硬编码字符串导致未来的维护偏差。
    *   地址栏历史记录上限（15条）目前已达到视觉极限，不建议继续增加，否则在低分辨率显示器上会导致面板底部溢出主窗口。
