# 列表视图星级与禁止图标与网格视图物理对齐 —— Modification_Plan-48.md

> 状态：已批准，执行中 / 已执行完成

## 1. 任务背景
为了保证“列表”、“网格”与“自适应”视图间视觉规范和交互逻辑的完全一致，用户提出将列表视图中的星级、禁止图标以及颜色标记胶囊背景与网格视图进行物理对齐。目前列表视图下的星级列（`TreeItemDelegate::paint`）使用独立的旧硬编码资源，色彩固定暗淡，且不支持胶囊背景，而星级尺寸与交互点击 Hitbox 也存在微小偏差。

## 2. 问题定位
- 渲染层：`src/ui/TreeItemDelegate.h` 中 `col == 2`（星级列）的绘制逻辑需要重构，改用统一的亮度对比度感知算法、统一的 SVG 图标、尺寸以及可选胶囊背景。
- 交互层：`src/ui/ContentPanel.cpp` 中的 `if (view == m_treeView)` 分支需要同步更新点击 Hitbox，使其与最新的绘制参数完美对位。

## 4. 详细解决方案

### 4.1 绘制端重构：`src/ui/TreeItemDelegate.h`
重构 `col == 2` 的绘制部分：
1. **提取元数据与色彩计算**：
   - 提取第 0 列的颜色标记 `colorHex`，并移植网格视图的 **感知亮度（Luminance）算法**：
     - 若 `colorHex` 为空（无色标）：星级和禁止图标使用亮灰色 `#CCCCCC`（实心/禁止）与 `#888888`（空心）。
     - 若 `colorHex` 不为空（有色标）：根据其 RGB 权值计算亮度，深色背景用白星（`#FFFFFF`），浅色背景用暗星（`#1A1A1A` / 亮度 < 0.5）。
2. **统一标准 SVG 资源文件**：
   - 禁止/清空图标：`"no_color"`
   - 实心星：`"star-svgrepo-com.svg"`（替换原有的 `"star_filled"`）
   - 空心星：`"star-rate-rating-outline-svgrepo-com.svg"`（替换原有的 `"star"`）
3. **统一物理排版尺寸**：
   - 禁止图标宽度 `banW = 14px`
   - 星级图标尺寸 `starSize = 18px`
   - 星级间距 `starSpacing = 1px`
   - 左侧起始偏移 `startX = option.rect.left() + 6`
4. **色标胶囊背景**：
   - 若存在 `colorHex`，计算整个胶囊包裹区（联合 `banRect` 和第 5 颗星的 `starRect`），在绘制图标前用 `drawRoundedRect` 绘制一层胶囊圆角矩形。

### 4.2 交互端重构：`src/ui/ContentPanel.cpp`
定位到 `ContentPanel::eventFilter` 方法中的 `view == m_treeView` 分支，更新判定参数：
1. **参数同步**：
   - 禁止图标尺寸 `banW = 14`
   - 星级尺寸 `starSize = 18`
   - 星间距 `starSpacing = 1`
2. **Hitbox 同步**：
   - 设定点击位置计算坐标，与 `TreeItemDelegate` 完全一致。
