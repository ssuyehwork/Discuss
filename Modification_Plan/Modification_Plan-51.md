# 列表视图星级物理紧贴对齐重构 —— Modification_Plan-51.md

> 状态：已批准，执行中 / 已执行完成

## 1. 任务背景
在网格视图（`GridItemDelegate`）中，评级元素的物理排列规格定义得极其明确：禁止图标与第一颗星之间存在 `2px` 间距（`banGap = 2`），而 5 颗星彼此之间是严格的 `0px` 无间距紧贴（`starSpacing = 0`）。为了在列表视图中也提供与网格视图绝对完美的 1:1 像素规格对齐，用户希望对列表视图下的星级绘制和鼠标点击命中区域（Hitbox）进行重新标定，将星间距改回紧贴模式。

## 2. 问题定位
- 渲染层：`src/ui/TreeItemDelegate.h` 的 `col == 2`（星级列）中星星绘制。
- 交互层：`src/ui/ContentPanel.cpp` 事件过滤器中的 `m_treeView` 鼠标位置命中检测。

## 4. 详细解决方案

### 4.1 绘制端精确对齐：`src/ui/TreeItemDelegate.h`
重写 `col == 2` 分支：
- 定义变量 `int banGap = 2;` 与 `int starSpacing = 0;`。
- 首星起点：`int starsStartX = startX + banW + banGap;`
- 在绘制胶囊和星级时：
  - 胶囊右边界：第 5 颗星的起点为 `starsStartX + 4 * starSize`（因 `starSpacing = 0`）。
  - 星级循环绘制：横坐标为 `starsStartX + i * starSize`。

### 4.2 交互端精确对齐：`src/ui/ContentPanel.cpp`
同步更新事件过滤器中 `m_treeView` 分支：
- 定义并使用完全一致的 `banGap = 2;` 与 `starSpacing = 0;`。
- 采用相同公式 `starsStartX = startX + banW + banGap;`
- 星级 Hitbox 范围橫坐标同样设为 `starsStartX + i * starSize`。

## 6. 实现准则与预警【核心】
1. **防范围溢出**：对齐后，整个星级区域在横向会比 2px 间距状态更紧凑、优雅，不仅完全保证在 120 像素的列宽中完美承载，也更契合网格视图规格。
2. **像素一致性**：对公式重新精算并进行零偏差对齐，杜绝任何 1 像素偏差带来的闪烁与误触。
