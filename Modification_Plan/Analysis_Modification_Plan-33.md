# Analysis_Modification_Plan-33: “其他属性”筛选分组的交互规范化重构

## 1. 现状问题深度剖析
此前的“其他属性”实现方案在交互逻辑和视觉规范上存在严重偏差，严重破坏了 ArcMeta 的极简主义设计语言：
1.  **位置倒置**：将辅助性的物理属性筛选置于面板顶部，干扰了用户对高频维度（如标签、颜色）的视觉捕捉。
2.  **控件违和**：在全局均采用 `StyledCheckBox` 的环境下，突兀地引入 QRadioButton（单选钮），导致操作习惯断层。
3.  **逻辑冗余**：设置“全部”单选选项。在复选框体系中，“均不勾选”即代表该维度不作为过滤条件（即显示全部），显式提供“全部”选项纯属画蛇添足。

## 2. 逻辑架构修正方案

### 2.1 分组位置沉底 (`src/ui/FilterPanel.cpp`)
在 `rebuildGroups()` 流程中，将“其他属性”（或建议更名为“高级属性”）分组的构建逻辑移至函数末尾，确保其始终位于面板的最底部，作为补充筛选手段。

### 2.2 控件回归复选框体系
废除所有单选控件，统一调用既有的 `addFilterRow()` 接口，生成与“标签”、“文件类型”等分组视觉一致的列表行：
*   **链接**：包含“有链接”、“无链接”两个复选框行。
*   **备注**：包含“有备注”、“无备注”两个复选框行。
*   **图像比例**：改为复选框列表，支持多选（例如可同时勾选“横图”与“方形”）。

### 2.3 智能互斥与隐含逻辑优化 (`src/ui/FilterPanel.h/cpp`)
为了在保持应用既有风格的同时处理互斥状态，设计以下逻辑：
1.  **移除“全部”项**：UI 上不再显示任何名为“全部”的按钮。
2.  **手动互斥判定**：
    *   在“链接”维度：若用户勾选“有链接”，则程序逻辑自动取消对“无链接”的勾选（反之亦然）。
    *   **隐含“全部”**：当“有链接”和“无链接”均未被勾选时，`FilterState` 相应标志位置为空/默认，代表不对此维度执行过滤。
3.  **多选合并**：对于“图像比例”，允许多个复选框同时处于勾选态，筛选器执行“或”逻辑匹配。

## 3. 具体修改建议

### 3.1 属性筛选逻辑重构
修改 `FilterState` 结构，移除三态枚举，改为更加直观的布尔或标志位：
```cpp
struct FilterState {
    // ...
    bool filterHasLink = false;
    bool filterNoLink  = false;
    bool filterHasNote = false;
    bool filterNoNote  = false;
    QStringList aspectRatios; // 存储勾选的比例类型，如 "landscape", "square"
};
```

### 3.2 UI 响应逻辑
在连接 `QCheckBox::toggled` 信号时，加入简单的互斥逻辑：
```cpp
connect(cbHasLink, &QCheckBox::toggled, this, [this, cbNoLink](bool on) {
    if (on) cbNoLink->setChecked(false); // 勾选“有”，自动取消“无”
    m_filter.filterHasLink = on;
    m_filter.filterNoLink = false;
    emit filterChanged(m_filter);
});
```

## 4. 验证要点
1.  **一致性验证**：检查“其他属性”行的高度、文字颜色、勾选框样式是否与“文件类型”等分组完全对齐。
2.  **重置逻辑**：点击面板顶部的“清除”图标（垃圾桶）时，确保下方所有新增的复选框均能正确回弹至未勾选状态。
3.  **隐含“全部”测试**：在不勾选链接和备注任何项的情况下，验证筛选结果是否包含所有类型的文件。
