# 列表视图星级与禁止图标间距统一物理对齐 —— Modification_Plan-49.md

> 状态：已批准，执行中 / 已执行完成

## 1. 任务背景
在列表视图下，禁止图标与第 1 颗星的间距因使用 `banRect.right() + 4` (QRect::right() 的 1px 像素偏差坑) 而导致实际间距达到了 3~4px，这与 5 颗星彼此之间 1px 的间距格格不入，视觉上显得极度不协调、甚至出现脱节感。为了提供顶级的 UI 物理一致性，用户希望用统一的绝对间距 `kSpacing = 2px` 重新计算并精确绘制所有相关图标，消除这些像素微小的视觉不一致。

## 2. 问题定位
- 渲染层：`src/ui/TreeItemDelegate.h` 第 182-243 行 `col == 2` 分支内，星星的 X 坐标计算依赖于 `banRect.right() + 4`，并且星星之间的间距写成了 `spacing = 1`。
- 交互层：`src/ui/ContentPanel.cpp` 中的 `m_treeView` 的 `eventFilter` 判定块内，Hitbox 具有完全相同的偏差。

## 4. 详细解决方案

### 4.1 绘制重构：`src/ui/TreeItemDelegate.h`
定义 `kSpacing = 2` 常量（2 像素），重写位置计算逻辑：
- 第 1 颗星的起始位置：`int starsStartX = startX + banW + kSpacing;`
- 星级循环绘制：使用 `starSize + kSpacing`。

### 4.2 交互重构：`src/ui/ContentPanel.cpp`
同步更新事件过滤器 `eventFilter`：
- 使用 `starsStartX = startX + banW + kSpacing;`
- 累加判定：使用 `starSize + kSpacing`。

## 6. 实现准则与预警【核心】
1. **防止像素漂移**：显式利用 `startX + banW + kSpacing` 代替 `banRect.right()` 计算，完美消除 Qt 矩形 right() 的 `left + width - 1` 的特有偏差，达到在各级行高下数学完美的像素级对齐。
2. **两端对位对齐**：确保渲染的逻辑与鼠标点击判定在物理模型上为同一个数学公式，实现完美的“指哪打哪”。
