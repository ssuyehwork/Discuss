# 复制粘贴逻辑架构分析

这是一个复用型提示，用于分析应用中明显由复制粘贴而成的代码片段，判断其架构适配性、逻辑配套性和运行稳定性。

## 何时使用
当你发现应用里某段代码看起来是直接复制粘贴过来的，导致逻辑架构不配套、适应性差、运行不理想、缺乏推导性时，使用此提示。

## 输入
- 选中的代码片段、相关函数、模块文件路径，或简要描述当前模块的行为
- 说明当前问题点，例如“逻辑重复”“职责混乱”“适配性弱”“运行不稳定”“用户体验差”

## 任务要求
请作为资深架构分析师：
1. 判断这段代码是否存在复制粘贴式实现（即重复逻辑、结构不一致、风格杂乱）
2. 分析当前逻辑架构是否配套、责任边界是否清晰、模块间适应性是否良好
3. 指出由于复制粘贴造成的具体问题，包括可维护性、扩展性、运行稳定性和用户使用体验
4. 提出合理的重构方向和优先级，建议哪些模块应先规范化、抽象或拆分
5. 不直接输出完整替换代码，重点给出分析结论和改进策略

## 输出格式
- 要点式结论
- 风险/问题清单
- 重构建议
- 适用场景说明

## 风格要求
- 中文表达
- 专业、简洁、针对性强
- 适用于 Qt/C++ 桌面应用
