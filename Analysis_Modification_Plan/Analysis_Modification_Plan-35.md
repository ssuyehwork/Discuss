# Analysis_Modification_Plan-35: 忠于视觉原型的文件大小过滤器重构

## 1. 偏差识别与纠正
针对此前方案中“自行创新”导致的布局偏差，现进行 100% 忠于用户图片的视觉对账：
*   **占位符错误**：此前方案未明确指定提示词，现强制修正为：左侧框提示“**最小**”，右侧框提示“**最大**”。
*   **布局错误**：此前建议的竖向堆叠方案严重偏离原型，现重归**水平流式布局**。
*   **元素间距**：两个输入框之间必须使用“**-**”符号连接，末尾衔接单位选择器，形成紧凑的工具链。

## 2. 局部视觉架构设计

### 2.1 组件水平封装 (`src/ui/FilterPanel.cpp`)
在“其他属性”分组内部，文件大小行应由一个独立的 `QHBoxLayout` 承载，严格按照以下顺序排列：
1.  **QLineEdit (Min)**：
    *   `setPlaceholderText("最小")`
    *   固定宽度（如 60px），6px 圆角。
2.  **QLabel**：
    *   `setText("-")`
    *   居中对齐，极简视觉。
3.  **QLineEdit (Max)**：
    *   `setPlaceholderText("最大")`
    *   固定宽度（如 60px），6px 圆角。
4.  **QComboBox (Unit)**：
    *   提供 KB/MB 选项，6px 圆角。
    *   视觉风格与输入框 100% 对齐。

### 2.2 视觉规范加固 (QSS)
为了消除“脑补”感，必须对这一行的所有控件应用统一的物理红线：
```css
/* 针对文件大小行内的子控件 */
QLineEdit#SizeLimitEdit, QComboBox#SizeUnitCombo {
    background: #2D2D2D;
    border: 1px solid #444444;
    border-radius: 6px;
    color: #CCCCCC;
    font-size: 11px;
}
QLineEdit#SizeLimitEdit:focus {
    border-color: #378ADD; /* 选中时的蓝色外边框，见图片细节 */
}
```

## 3. 逻辑同步
*   **占位符行为**：当用户点击输入框时，提示词“最小”/“最大”自动消失（Qt 默认行为），确保输入引导清晰。
*   **容器适配**：虽然该组件内部为横排，但由于其各元素宽度已固定且经过极致压缩，即便在侧边栏较窄的情况下也能完美容纳，不影响外层 `FilterPanel` 的整体纵向滚动规范。

## 4. 验证要点
1.  **视觉 1:1 还原**：打开截图与实时界面进行重叠比对，确保占位符文字内容、控件排列顺序、圆角弧度完全一致。
2.  **焦点状态验证**：点击“最小”输入框，验证其是否会出现图中所示的蓝色高亮边框（如图片 1 部分所示）。
3.  **单位联动测试**：切换 MB/KB 时，验证后台 `minSize`/`maxSize` 字节计算的准确性。
