# Analysis and Modification Plan - 标签管理界面布局优化（统一 20px 间距与单分割线原则）

## 1. 现状剖析
在当前的“标签管理”视图（`TagManagerView`）中：
- 标题下方存在多余的分割线（`border-bottom`），导致视觉混乱且违背了“一组一割”的原则。
- 标题、标签按钮与分割线之间的垂直间距不统一且过于拥挤。

## 2. 解决方案建议（已达成共识）

### 2.1 彻底清理多余线条
- **修正点**：移除 `groupTitle`（如“#”、“A”）的 `border-bottom` 样式。
- **结果**：标题下方将呈现干净的留白，不再有线条干扰。

### 2.2 落实“一组一割”原则
- **分割线位置**：每一组标签（标题 + 标签列表）仅在最底部保留**唯一**一条分割线。
- **实现方式**：在组容器 `groupWidget` 的底部设置 `border-bottom: 1px solid #333`。

### 2.3 垂直间距标准化（20px）
- **标题与标签的间距**：设定为 **20 像素**（通过 `vLayout->setSpacing(20)` 实现）。
- **标签与底部分割线的间距**：设定为 **20 像素**（通过 `vLayout->setContentsMargins(0, 0, 0, 20)` 实现）。

### 2.4 高度计算公式同步（用于 adjustFlowHeights）
- **计算公式**：`setFixedHeight(16 + 20 + h + 20)` (即 `h + 56`)
  - `16`: 标题固定高度。
  - `20`: 标题与标签容器的垂直间隙。
  - `h`: 标签列表的流式动态渲染高度。
  - `20`: 标签列表底部到组分割线的垂直间隙。

## 3. 执行规范
- 严格遵循“标题下方不准有线”的要求。
- 确保 20px 的间距在全量分组中物理一致。
