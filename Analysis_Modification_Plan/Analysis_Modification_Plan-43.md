# Analysis and Modification Plan - AGENTS.md 准则增补（杜绝脑补另创专版）

## 1. 现状分析
在此前的任务中，AI（Jules）在处理 `MainWindow` 搜索框清除按钮时，违背了项目已有的“原生优先”惯例，擅自脑补并创建了“自定义 Action 模式”。这种行为直接导致了视觉系统的不统一，并触碰了项目管理的红线。

## 2. AGENTS.md 增补条文建议
为了从制度上锁死 AI 的脑补空间，建议在 `AGENTS.md` 中新增以下内容：

---

### # 4. 严禁脑补与另创标准（最高指令）

#### 4.1 唯一路径原则
- 当用户指定或项目已存在某种技术实现路径（如“Qt 原生接口”）时，AI **严禁**以任何借口（美化、优化、解耦等）脑补出第二套实现方案。
- **强制对标**：新功能的实现必须物理级对标旧有模块的成熟方案。严禁在 A 页面用原生，B 页面用自定义。

#### 4.2 杜绝“顺手优化”
- 严禁在修改代码时进行任何“顺手”的逻辑重构或样式微调。
- **唯一操作权**：AI 的操作权仅限于“达成共识”的方案内容。超出方案哪怕 1 像素、1 行代码的修改均被视为严重违规。

#### 4.3 申报备案制（Declaration Log）
- 如果 AI 认为某种原生方案或指定方案存在缺陷，**只准动笔，不准动手**。
- 必须将想法登记到 `Declaration_Log.md` 中备案，并在方案中如实记录，等待用户下令。未获明确指令前，必须执行原定方案。

#### 4.4 行为自省要求
- 在每次制定方案前，AI 必须自问：“我是否在脑补另创？”、“我是否在背离已有标准？”。

---

## 3. 预期效果
1. 强制 AI 回归“执行者”本位，剥夺其在未经授权下的“设计师”角色。
2. 确保 `MainWindow` 搜索框与 `FilterPanel` 的清除按钮等 UI 元素实现方案的绝对物理一致性。

## 4. 执行规范提醒
- 修改 `AGENTS.md` 时，必须一字不差地将上述准则植入，不得简化。
- 发现任何潜在冲突，立即记录至 `Declaration_Log.md`。
