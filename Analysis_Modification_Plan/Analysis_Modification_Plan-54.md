# 双轨制架构分析与“范围感知”搜索方案 —— Analysis_Modification_Plan-54.md

## 1. 逻辑架构分析：双轨制数据驱动模式
目前的底层核心逻辑架构分为两种互斥模式，通过 UI 顶部的 **1px 蓝色提示线 (Focus Line)** 进行物理视觉反馈：

### 1.1 侧边栏分类模式 (DB 类数据库)
- **数据来源**：SQLite 数据库 (`category_items` 表)。通过逻辑关联（FID）组织文件，支持一文件多分类。
- **触发机制**：点击侧边栏分类项，`ContentPanel` 发出 `dataSourceChanged("category")` 信号。
- **视觉表现**：`CategoryPanel` 顶部出现蓝色提示线。
- **逻辑语义**：当前内容容器展示的是“逻辑视图”，即数据库中打标关联的数据。

### 1.2 磁盘路径动态模式 (实时 I/O 扫描)
- **数据来源**：物理磁盘文件系统。通过实时 I/O 扫描（MFT 或 QDir）获取目录结构。
- **触发机制**：在目录树中导航或双击文件夹，`ContentPanel` 发出 `dataSourceChanged("nav")` 信号。
- **视觉表现**：`NavPanel` 顶部出现蓝色提示线。
- **逻辑语义**：当前内容容器展示的是“物理视图”，即磁盘上真实存在的物理层级。

## 2. 搜索框重构方案：基于“范围感知 (Scope-Aware)”的焦点联动
**现状问题**：当前的搜索框（`m_searchEdit`）仅支持从全局 DB 数据库检索，无视当前的焦点模式（蓝色提示线位置），违背了“所见即所搜”的设计理念。

### 2.1 重构逻辑目标
搜索行为必须实时匹配当前的焦点线状态，实现“范围感知”：

| 当前焦点位置 | 搜索模式 | 底层实现建议 |
| :--- | :--- | :--- |
| **侧边栏 (DB 模式)** | **分类内搜索** | 调用数据库接口，增加 `WHERE category_id = ?` (含子类) 过滤条件。 |
| **目录导航 (I/O 模式)** | **路径内搜索** | 针对当前物理路径执行实时 MFT 匹配或递归文件系统扫描。 |
| **无焦点 (如标签管理)** | **特定模式搜索** | 维持当前标签管理器的检索逻辑。 |
