# 地址栏虚拟路径编辑态显示异常分析 —— Analysis_Modification_Plan-76.md

## 1. 任务背景
用户反馈在“此电脑”视图下，点击地址栏进入编辑模式时，显示的文本是内部协议名 `computer://`，而非用户友好的中文名称“此电脑”。

## 2. 深度诊断结论

### 2.1 状态切换逻辑缺失 (State Transition Void)
在 `AddressBar.cpp` 中，负责将面包屑（Breadcrumb）切换为输入框（LineEdit）的函数是 `onBreadcrumbBlankClicked`。

* **缺陷代码**:
```cpp
void AddressBar::onBreadcrumbBlankClicked() {
    // 错误点：此处直接使用了原始路径，并套用了物理路径转换函数
    m_pathEdit->setText(QDir::toNativeSeparators(m_currentPath)); 
    m_pathStack->setCurrentWidget(m_pathEdit);
    // ...
}
```
* **分析**: `m_currentPath` 在“此电脑”视图下存储的是逻辑协议字符串 `computer://`。调用 `QDir::toNativeSeparators` 会将其转换为物理风格的 `computer:\\`，但它无法识别该路径是一个需要本地化翻译的虚拟节点。

### 2.2 逻辑不一致性
虽然 `setPath` 函数中包含了翻译逻辑：
```cpp
void AddressBar::setPath(const QString& path) {
    QString displayPath = (path == "computer://") ? "此电脑" : QDir::toNativeSeparators(path);
    m_pathEdit->setText(displayPath);
    // ...
}
```
但在用户**点击空白处主动触发编辑态**时，系统绕过了 `setPath`，直接操作 `m_pathEdit->setText`，导致之前的翻译逻辑失效。

## 3. 解决方案 (资深程序员视角)

### 3.1 引入路径翻译中枢 (Path Translation Engine)
建议在 `AddressBar` 中封装一个私有方法 `getDisplayPath(const QString& rawPath)`，统一处理物理路径归一化与虚拟协议翻译。

### 3.2 修正编辑触发逻辑
修改 `onBreadcrumbBlankClicked`，确保在进入编辑模式时同样调用翻译逻辑：
```cpp
void AddressBar::onBreadcrumbBlankClicked() {
    // 方案：调用统一的显示路径转换函数
    QString displayPath = (m_currentPath == "computer://") ? "此电脑" : QDir::toNativeSeparators(m_currentPath);
    m_pathEdit->setText(displayPath);
    
    m_pathStack->setCurrentWidget(m_pathEdit);
    m_pathEdit->setFocus();
    // ...
}
```

### 3.3 补全回车跳转后的回滚逻辑
在 `returnPressed` 的 `else` 分支中，当输入非法路径并试图恢复显示时，也应同步应用翻译。

## 4. 结论
这是一个典型的**逻辑路径（Logical Path）与显示路径（Display Path）未完全解耦**的问题。在 UI 交互的状态机转换中（从展示态到编辑态），漏掉了对特殊协议名的映射检查。
